--> Inheritance
Passing the properties of parent class to the child class
Put the parent class name in child class () --> 'childclassname(parentclassname)''
import the parent class in child class file --> 'from <file_name> import <classname_to_import>''
If you do not have default constructor then make sure to import parent constructor into child constructor, if parent class has default constructor then no need to call parent constructor into child constructor

--> Exception Handling
1). 'raise Exception:(<message_to_display>)' --> when condition gets failed then it raise the exception with given message
2). 'assert(<true_condition>)' --> when given condition gets False then it raises the assertion error, condition should be true here (true condition-- should match to required criteria)
       --> for eg. assert Hi == Hello, "Given two strings does not matches"
           -- Here, we asserting the condition like Hi == Hello, it is not matching so it will raise an assertion error with the the message which we given after ','
           -- This message only prints when assertion is failed, if assertion is passed then message will not print
3). 'try:'-'except:'(Catch) Mechanism
        --> Whenever a condition does not satisfied then the test case failed, but if you want to go next block without failure then use 'try_except' block mechanism
        --> The test case which is gonna to be failed, put that in 'try:' block and followed by 'except:' block where we can display your own customized reason of failure of try block and further test suite execution will go on
        --> If you don't want to display your own customized failure message -- use 'Exception' keyword, this key has it's own message given by python
4). 'finally: --> This keyword is associated with try except mechanism, this finally block always executes whether we get error in try block or not
              --> Even if whole test suite execution fails, still finally block will be execute successfully
              --> Significance => To clear the data/junk data
                               => eg. When a test case failed, so some data gets stored and you need fresh or no data for new execution so use finally block, where you implement the step to clean up the data
